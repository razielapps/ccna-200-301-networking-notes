Topic 6: Introduction to VPN

A Virtual Private Network (VPN) is a technology that creates a secure, encrypted connection over a less secure network, typically the public Internet. Think of it as building a secure, private tunnel through a public highway. This allows remote users or branch offices to securely access a central network as if they were directly connected to it locally.

The primary benefits of VPNs are:
Cost Savings: Eliminates the need for expensive dedicated WAN lines by using the Internet.
Security: Protects data confidentiality and integrity as it travels over untrusted networks.
Flexibility: Allows employees to work securely from anywhere.

There are two main types of VPNs:

1. Remote Access VPN: This type is for individual users connecting to a central network. A common example is an employee using VPN client software on their laptop to connect to the corporate network from a coffee shop or home. The user's device authenticates to a VPN concentrator at the headquarters, and all their traffic is securely tunneled.

2. Site-to-Site VPN: This type connects entire networks together. For example, it connects a branch office network to a headquarters network over the Internet. Site-to-site VPNs are typically established between routers or firewalls. There are two sub-types:
   Intranet VPN: Connects networks within the same organization.
   Extranet VPN: Connects the networks of different organizations, like a company and its supplier.

The most common protocol suite used to create VPNs is IPsec (Internet Protocol Security). IPsec provides a framework for securing IP communications by authenticating and encrypting each IP packet in a data stream. It operates at the network layer, protecting everything above it.

IPsec has two main components:
1. Authentication Header (AH): Provides authentication and integrity (ensuring the packet hasn't been tampered with) but does not provide encryption.
2. Encapsulating Security Payload (ESP): Provides authentication, integrity, and confidentiality (encryption). ESP is much more commonly used.

To establish an IPsec VPN, two phases must be completed:
IKE Phase 1: The two peers authenticate each other and negotiate a secure, authenticated channel. This is used to protect the negotiations for the actual data tunnel.
IKE Phase 2: The peers negotiate the security parameters (like the encryption algorithm) for the actual data tunnel that will carry user traffic.

Another popular VPN technology is SSL VPN (Secure Sockets Layer VPN), which uses the same SSL/TLS protocols used for secure websites (HTTPS). It is very common for remote access VPNs because it can often run directly in a web browser without dedicated client software.

While VPNs provide strong security, they are not a single solution. A VPN protects data in transit, but other security measures like firewalls, intrusion prevention systems, and endpoint security are still needed to protect the networks at each end of the tunnel.

Assignment 6: Configuring a Basic Site-to-Site IPsec VPN in Packet Tracer

Objective: To understand the components and configuration of a simple site-to-site VPN. (Note: Packet Tracer's VPN capabilities are limited but suitable for demonstrating concepts).

1. In Packet Tracer, create a topology with two routers, R1 and R2, representing two branch offices. Connect each router to a separate "ISP" cloud to simulate the Internet. Use the "Cloud" device in Packet Tracer and connect via serial interfaces. Assign public IP addresses to the router interfaces connected to the clouds (e.g., R1: 203.0.113.1/24, R2: 198.51.100.1/24). Configure static default routes on R1 and R2 pointing to the ISP clouds.
2. Configure the internal LANs for each router.
   R1: G0/0 - 192.168.10.1/24. Connect a PC with IP 192.168.10.10.
   R2: G0/0 - 192.168.20.1/24. Connect a PC with IP 192.168.20.10.
3. On R1, navigate to the VPN section in the GUI (GUI is easier for VPN in PT).
   Go to the 'Services' tab > VPN > Site-to-Site.
   Set Connection Type to 'IPSec'.
   Set Local Network: 192.168.10.0 / 255.255.255.0
   Set Remote Network: 192.168.20.0 / 255.255.255.0
   Set Remote Gateway: 198.51.100.1 (R2's public IP)
   Set Pre-shared Key: mySecretKey123
   Click 'Add' and then 'Save'.
4. On R2, perform the mirror configuration.
   Go to 'Services' > VPN > Site-to-Site.
   Set Local Network: 192.168.20.0 / 255.255.255.0
   Set Remote Network: 192.168.10.0 / 255.255.255.0
   Set Remote Gateway: 203.0.113.1 (R1's public IP)
   Set Pre-shared Key: mySecretKey123
   Click 'Add' and 'Save'.
5. The VPN tunnel should establish. To test, go to the PC on the 192.168.10.0 network and ping the PC on the 192.168.20.0 network. The ping should be successful. The traffic is being encrypted by R1, sent over the simulated Internet, and decrypted by R2.
6. Use the command `show crypto isakmp sa` on R1 or R2 to view the Phase 1 security associations. Use `show crypto ipsec sa` to view the Phase 2 security associations and see statistics for encrypted and decrypted traffic.

Glossary for Topic 6:
VPN: A secure, logical network created over a public network.
IPsec: A protocol suite for securing IP communications.
Site-to-Site VPN: A VPN that connects two fixed networks.
Remote Access VPN: A VPN for individual users to connect to a network.
Tunneling: The process of encapsulating one packet within another for secure transmission.
